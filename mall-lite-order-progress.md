## 6. 技术决策记录

| 日期 | 决策项 | 当前决策 | 原因 | 后续是否可调整 |
| --- | --- | --- | --- | --- |
| 2026-07-23 | 架构形态 | Spring Boot 单体模块化 | 更适合求职项目展示完整业务闭环，避免微服务复杂度过早分散重点 | 可调整 |
| 2026-07-23 | Java 版本 | Java 21 | 当前仓库根 POM 已使用 Java 21，保持一致 | 不建议调整 |
| 2026-07-23 | 数据库 | MySQL 8 | 符合 Java 后端岗位常见技术栈，适合展示事务、索引和 SQL 优化 | 不建议调整 |
| 2026-07-23 | 缓存 | Redis | 用于商品详情缓存、热点数据和分布式锁扩展 | 不建议调整 |
| 2026-07-23 | 消息队列 | RabbitMQ | 用于订单异步通知和超时取消，当前仓库已有相关练习模块 | 不建议调整 |
| 2026-07-23 | 超时取消方案 | RabbitMQ TTL + DLX | 不依赖延迟插件，本地环境更容易复现 | 可调整 |
| 2026-07-23 | 防超卖第一方案 | MySQL 条件更新 | 简单、可靠，适合作为第一版库存一致性方案 | 可扩展 Redisson |
| 2026-07-23 | ORM 框架 | MyBatis-Plus | 在保留 MyBatis 思路的同时减少基础 CRUD 样板代码，更适合快速完成项目闭环 | 可调整 |
| 2026-07-23 | 表命名规范 | `mall_*` 前缀 | 避免使用 `user`、`order` 等数据库保留字或高冲突名称 | 不建议调整 |
| 2026-07-23 | 本地依赖端口 | MySQL `13306`、Redis `16379`、RabbitMQ `5673/15673` | 避免和仓库已有通用 Docker 服务默认端口冲突 | 可调整 |
| 2026-07-23 | MyBatis-Plus ID 生成器 | 固定 workerId/datacenterId 为 `1/1` | 避免受限环境中自动探测网卡导致测试噪声或失败 | 可调整 |
| 2026-07-23 | 认证方案 | Spring Security + JWT Bearer Token | 单体后端项目足够清晰，适合展示无状态认证和接口保护 | 后续可扩展刷新令牌 |
| 2026-07-23 | 密码存储 | BCrypt | 避免明文或弱哈希存储密码，符合后端认证基础安全要求 | 不建议调整 |
| 2026-07-23 | 默认访问控制 | 白名单放行，其余接口认证 | 以后新增订单、支付等接口默认受保护，避免漏配鉴权 | 不建议调整 |
| 2026-07-23 | 商品浏览权限 | 分页和详情公开，创建/上下架需登录 | 浏览商品符合电商场景；管理商品属于写操作，需要认证 | 后续可加管理员角色 |
| 2026-07-23 | 商品状态 | `1` 上架，`0` 下架 | 与数据库脚本保持一致，订单链路后续只允许购买上架商品 | 不建议调整 |
| 2026-07-23 | 商品分页 | MyBatis-Plus `Page` + PaginationInnerInterceptor | 避免手写分页 SQL，并为后续列表查询保持统一分页响应 | 不建议调整 |

## 7. 当前风险与问题

| 问题 | 影响 | 当前处理方式 |
| --- | --- | --- |
| 商品详情缓存尚未实现 | 商品查询目前仍直接访问数据库 | 下一步接入 Redis 缓存商品详情 |

| 技术点较多 | 容易做成半成品 | 先跑通订单主链路，再逐步接入缓存、MQ 和并发强化 |
| 尚无压测数据 | 简历缺少量化结果 | 后期使用 JMeter、wrk 或 ApacheBench 补充真实数据 |
| 当前测试主要覆盖上下文和基础访问控制 | 尚未覆盖数据库读写场景 | 商品模块或用户模块集成测试阶段引入测试数据库或本地 MySQL |
